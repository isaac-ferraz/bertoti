# 1 - O SOFTWARE É INTANGÍVEL

> O texto bate em algo que eu acho bem verdadeiro: a engenharia de software ainda está amadurecendo como disciplina. A comparação com engenharia aeronáutica é certeira — um avião que cai por erro de cálculo é imediato e visível, enquanto um bug de software pode passar anos sem ser notado ou causar danos silenciosos. O que me chamou atenção foi a honestidade de admitir que a área ainda não tem o mesmo rigor das engenharias tradicionais, mas que precisa caminhar nessa direção conforme o software se torna cada vez mais crítico no dia a dia das pessoas. No fundo, é um convite pra levar a profissão mais a sério — o que faz todo sentido num mundo onde um sistema falho pode comprometer desde uma transação bancária até um equipamento médico.


# 2 - ENGENHARIA DE SOFTWARE = Programação + tempo, escalabilidade e trade-offs

> Esse trecho traz uma definição bem mais madura do que é engenharia de software — não é só escrever código, é pensar no que acontece com esse código ao longo do tempo. A ideia de "programação integrada ao tempo" é simples, mas poderosa: qualquer sistema vai precisar mudar, crescer e eventualmente ser descontinuado, e ignorar isso desde o início é o que transforma projetos promissores em pesadelos de manutenção. Os três pilares que o texto apresenta — tempo, escala e trade-offs — são exatamente o tipo de coisa que separa quem só escreve código de quem realmente pensa em construir software que dure. É uma perspectiva que faz bastante sentido especialmente pra quem está começando na área, porque é fácil focar só no "funciona agora" e esquecer do "vai continuar funcionando daqui a dois anos, com o dobro de usuários e uma equipe diferente".

# 3 - Requisitos não funcionais

> Escalabilidade: o software tem o grande poder , se for bem feito, de suportar um grande número de acessos. É muito importante ter essa preocupação com o software.

> Segurança: É importante estruturar um software para que ele seja seguro. Lugares como um banco, por exemplo, precisam ter confiar que o software será um lugar seguro para colocar dados confidenciais. 

> Recuperabilidade: o potencial que o software possui de resgatar algum arquivo ou outra coisa semelhante caso haja alguma perda ou alteração não intencional. Isso é um requisito muito importante nele.

> Performance: O sistema consegue alcançar uma performance muito boa num tempo necessário e usando apenas recursos disponíveis. Isso é um grande diferencial nele, pois mostra a velocidade que um software tem.

> Deploy: A habilidade que o software tem de ter um deploy de maneira fácil e eficiente. Tornando-o ágil e melhor.


# 4 - Trade-offs (negociação entre requisitos não funcionais)

> Ryzen (alta performance) x intel (eficiência energética)

> Playstation (desempenho gráfico) x Xbox (potência de hardware)

> Dados compactados (mais leve) x dados descompactados (leitura mais rápida)
